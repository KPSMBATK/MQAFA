Arkitektur Sistem & Integrasi Google Drive

Selaras dengan keperluan spesifikasi projek, sistem ini direka untuk menghubungkan penyerahan borang muat naik terus ke Google Drive tanpa mendedahkan kata laluan peribadi di web awam.

Logik Teknikal (Di Sebalik Tabir)

  1. Muka Hadapan (Frontend): Pensyarah menekan butang "Hantar Pindaan". Laman web membungkus fail dokumen tersebut dan menghantarnya ke pelayan (backend).
  2. Akaun Perkhidmatan (Service Account): Pada pelayan, satu "Akaun Perkhidmatan Google" diwujudkan yang bertindak sebagai pembantu robot digital. Robot ini diberikan kebenaran khusus (Google Drive API v3) untuk melihat dan menyunting folder tertentu (cth: "MQA Workshop Submissions 2026").
  3. Penyerahan (API Call): Pelayan (Node.js/Express atau Google Apps Script) menyerahkan fail kepada robot menggunakan API tersebut.
  4. Penyimpanan: Robot memuat naik fail ke dalam Drive dan secara automatik menamakan semula fail bagi tujuan susunan sistematik (cth: [BAK3283] - Amendment - Dr. Sarah.pdf).
  5. Resit Penerimaan: Google Drive memulangkan pautan fail. Laman web memaparkan status "Selesai" (Hijau) pada papan pemuka pensyarah, manakala pautan fail direkodkan ke dalam Google Sheet untuk rujukan pentadbir.

✨ Integrasi API Gemini (AI Assistant)

Aplikasi ini disepadukan dengan model gemini-2.5-flash-preview-09-2025 menggunakan strategi pemulihan tangkas (exponential backoff) sehingga 5 percubaan sekiranya berlaku had kadar akses (rate limits). Ia melakukan pemprosesan bahasa semulajadi secara langsung bagi membantu pensyarah menstruktur semula kurikulum mereka dengan pantas.

🛠️ Nota Prototaip:

Aplikasi interaktif ini pada masa ini menjalankan simulasi (mock) penyerahan fail untuk mendemonstrasikan aliran UI/UX dengan sempurna di dalam penyemak imbas (browser) anda, manakala bahagian Kecerdasan Buatan (Gemini AI) berfungsi secara langsung untuk memudahkan perancangan pengajaran anda.
